3. Key Observations

3.1 No Server-Side Processing

All processing activities occur locally on the user's device. MidMightBit Games does not operate any server infrastructure. There is no server-side database, no cloud storage, and no backend API. The App communicates directly with public blockchain infrastructure and third-party price APIs.

3.2 No Data Processors

As defined under GDPR Article 28, no data processors are engaged. Third-party RPC providers and price APIs receive only the minimum data necessary for their function (wallet address for RPC queries, token identifiers for price queries) and are independent controllers of any data they process (such as IP addresses in server logs).

3.3 Cross-Border Transfers

When making RPC calls, the user's wallet public address is transmitted to:

  • Solana public RPC: Decentralised infrastructure, various jurisdictions
  • Helius RPC: United States
  • CoinGecko API: Various jurisdictions (IP address only)
  • Jupiter Price API: Various jurisdictions (IP address only)

The wallet public address is already publicly available on the Solana blockchain. For EU/UK users, the transfer is necessary for the provision of the service at the user's request (GDPR Article 49(1)(b)).

3.4 Data Protection Impact Assessment (DPIA)

A formal DPIA is not required because:

  • The processing does not involve systematic and extensive profiling with significant effects
  • No special category data is processed at large scale
  • No systematic monitoring of publicly accessible areas occurs
  • The processing is minimal, local, and under the user's direct control

This determination is consistent with the Article 29 Working Party guidelines on DPIAs (WP248 rev.01) and the OAIC's guidance on privacy impact assessments under the Australian Privacy Act.
